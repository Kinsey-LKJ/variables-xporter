# 变量导出 📤

变量导出是 Variables Xporter 的核心功能，它能够将 Figma 中的设计变量转换为开发环境可用的代码。支持两种主要的导出模式：CSS 变量和 Tailwind CSS 配置。

## 导出模式

### 1. CSS 变量模式
这种模式会生成标准的 CSS 自定义属性文件：

```css
:root {
  --colors-primary: #c03939;
  --spacing-container: 1rem;
  --font-size-heading-1: 2.5rem;
}
```

### 2. Tailwind CSS 模式
这种模式会生成两个文件：
- `tailwind.config.js` - Tailwind CSS 配置
- `global.css` - CSS 变量定义

```js
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: 'rgb(var(--colors-primary))'
      },
      spacing: {
        container: 'var(--spacing-container)'
      }
    }
  }
}
```

## 智能优化

### 1. 颜色值优化
自动将颜色值转换为最优格式：
- RGB 通道格式，支持 Tailwind 的透明度修饰符
- 保留原始颜色格式用于 CSS 变量模式

### 2. 变量合并
智能合并相关变量，例如将字体相关变量组合成 Tailwind 的字体配置：

```js
fontSize: {
  'heading-1': ['2.5rem', {
    lineHeight: '1.2',
    fontWeight: '600'
  }]
}
```

### 3. 主题模式支持
自动处理不同主题模式下的变量值：

```css
:root {
  --colors-primary: var(--colors-primary-light);
}

.dark {
  --colors-primary: var(--colors-primary-dark);
}
```

## 使用建议

1. **选择合适的导出模式**
   - 使用 Tailwind CSS？选择 Tailwind 模式获得最佳集成
   - 需要更灵活的方案？选择 CSS 变量模式

2. **检查导出结果**
   - 验证变量名称是否符合预期
   - 确认变量值格式是否正确
   - 测试主题切换功能

3. **保持命名一致性**
   - 遵循变量命名规范
   - 使用语义化的命名
   - 保持层级结构清晰

<Callout type="info" emoji="💡">

</Callout>
