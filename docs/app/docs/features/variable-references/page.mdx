# 变量引用 🔗

变量引用是 Variables Xporter 最强大的特性之一，它不仅允许你在不同的变量集合之间建立引用关系，更是构建复杂多维设计系统的基石。通过变量引用，你可以从简单的主题切换逐步扩展到完整的多模式变量系统。

## 什么是变量引用？

变量引用允许一个变量使用另一个变量的值，这种引用可以：
- 跨越不同的变量集合，实现模块化管理
- 建立多层级的引用链，支持复杂的变量系统
- 最终追溯到原始类型的值（颜色、数字、字符串等）
- 在不同模式间灵活切换，保持系统的一致性

## 使用场景

让我们通过一个循序渐进的例子来看看如何使用变量引用构建强大的多维变量系统：

### 1. 基础的主题切换
首先从简单的明暗主题切换开始：
```plaintext
Design Tokens (主集合)
└── colors/primary/DEFAULT ────────────┐
                                       │
Theme Modes (主题模式集合)               │
└── colors/primary/DEFAULT ◄───────────┘
    └── modes
        ├── light: #E55517
        └── dark:  #EE6D21
```

### 2. 加入品牌主题
然后扩展支持多品牌：
```plaintext
Design Tokens (主集合)
└── colors/primary/DEFAULT ───────────┐
                                     │
Theme Modes (主题模式集合)           │
└── colors/primary/DEFAULT ◄─────────┼────┐
    └── modes                        │    │
        ├── light ◄─────────────────┐│    │
        └── dark  ◄─────────────────┼┘    │
                                    │     │
Brands Modes (品牌模式集合)         │     │
└── colors/primary                  │     │
    ├── light ──────────────────────┘     │
    │   ├── zhiyun: #E55517              │
    │   └── fiveray: #DA5959             │
    └── dark ────────────────────────────┘
        ├── zhiyun: #EE6D21
        └── fiveray: #C03939
```

### 3. 扩展到响应式设计
最后加入响应式支持：
```plaintext
Design Tokens (主集合)
└── font/hero/size ──────────────────┐
                                     │
Device Modes (设备模式集合)          │
└── font/hero/size ◄─────────────────┘
    └── modes
        ├── default: font/size/2xl (24px)
        └── min-width:1024px: font/size/3xl (30px)
```

通过这种方式，我们可以逐步构建一个支持多个维度的变量系统，每个维度都可以独立管理和扩展。查看[多模式变量管理](/docs/organizing-your-variables/multi-mode)了解更多复杂场景的处理方法。

## 最佳实践

1. **[使用单一出口原则](/docs/organizing-your-variables/multi-mode#single-export-principle)**
   - 所有变量访问通过主集合进行
   - 在主集合中建立引用关系
   - 避免直接使用其他集合的变量

2. **保持引用链清晰**
   - 避免过深的引用层级
   - 使用有意义的变量名
   - 记录关键引用关系

3. **合理组织变量集合**
   - 每个集合专注于特定功能
   - 明确集合之间的依赖关系
   - 避免循环引用

<Callout type="info" emoji="💡">
  使用变量引用时，Variables Xporter 会自动追踪并解析完整的引用链，你只需要在主集合中使用变量，无需关心背后的引用关系。
</Callout>
