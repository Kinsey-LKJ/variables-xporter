# 多模式支持 🎨

多模式支持是 Variables Xporter 最强大的特性之一，它让你能够构建一个真正灵活和可扩展的设计系统。从简单的明暗主题切换到复杂的多维变量管理，Variables Xporter 都能优雅地处理。查看[多模式变量的组织建议](/docs/organizing-your-variables/multi-mode)了解详细信息。

## 打造强大的多维变量系统

多模式支持不仅仅是简单的主题切换，它是一个完整的变量管理解决方案，让你能够：

### 多维度变量管理
- **明暗主题**：为不同光照环境优化的界面样式
- **品牌主题**：统一管理多个品牌的设计语言
- **响应式设计**：根据设备特性自适应的布局系统
- **交互状态**：细致的状态切换效果（Normal/Hover/Active）
- **紧凑模式**：为不同使用场景优化的布局密度

### 灵活的组合方式
- 支持模式间的任意组合，如"暗色主题下的品牌 A 紧凑模式"
- 每个维度都可以独立管理和扩展
- 通过单一出口原则保持系统的清晰和可维护

## 工作原理

### 1. 模式定义
在 Figma 中创建不同的模式：
```plaintext
Theme Modes（主题模式）
├── light
└── dark

Brand Modes（品牌模式）
├── default
├── brand-a
└── brand-b
```

### 2. 变量映射
为每个模式设置对应的变量值：
```css
/* 明暗主题示例 */
:root {
  --colors-primary: var(--colors-primary-light);
}

.dark {
  --colors-primary: var(--colors-primary-dark);
}

/* 品牌主题示例 */
.brand-a {
  --colors-primary: var(--colors-brand-a-primary);
}
```

### 3. 自动追踪
Variables Xporter 会自动：
- 识别所有可用的模式
- 生成对应的 CSS 变量
- 处理模式切换的逻辑

## 使用建议

1. **模式命名规范**
   - 使用清晰的模式名称
   - 遵循一致的命名模式
   - 避免特殊字符

2. **模式组织**
   - 按功能分组模式
   - 保持模式层级简单
   - 记录模式用途

3. **值的管理**
   - 确保所有模式都有对应的值
   - 使用合适的默认值
   - 测试模式切换效果
