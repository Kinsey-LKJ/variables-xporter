# CSS 变量导出

选择 CSS Variables 导出方式时，Variables Xporter 会生成一个包含所有 CSS 变量定义的 CSS 文件。这种方式更加简单和灵活，适合不使用 Tailwind CSS 的项目。

## 特点

### 1. 简单直观
直接输出标准的 CSS 自定义属性：

<CodePreview
  before={`// Figma 变量
colors/
  primary: #C03939
  primary-foreground: #FFFFFF
typography/
  heading-1/
    size: 40px
    line-height: 1.2`}
  after={`:root {
  --colors-primary: #c03939;
  --colors-primary-foreground: #ffffff;
  --font-size-heading-1: 2.5rem;
  --line-height-heading-1: 1.2;
}`}
  language="css"
/>

### 2. 灵活的使用方式
可以在任何 CSS 规则中使用这些变量：

```css
.button {
  background-color: var(--colors-primary);
  color: var(--colors-primary-foreground);
  font-size: var(--font-size-heading-1);
  line-height: var(--line-height-heading-1);
}
```

### 3. 主题模式支持
支持使用 CSS 类切换主题：

```css
:root {
  --colors-primary: var(--colors-primary-light);
}

.dark {
  --colors-primary: var(--colors-primary-dark);
}
```

## 实际使用场景

### 1. 响应式设计
```css
/* 基础变量 */
:root {
  --spacing-container: 1rem;
}

/* 媒体查询中修改变量值 */
@media (min-width: 768px) {
  :root {
    --spacing-container: 2rem;
  }
}

/* 使用变量 */
.container {
  padding: var(--spacing-container);
}
```

### 2. 组件变体
```css
/* 定义基础变量 */
.button {
  --button-bg: var(--colors-primary);
  --button-text: var(--colors-primary-foreground);
  background: var(--button-bg);
  color: var(--button-text);
}

/* 创建变体 */
.button.secondary {
  --button-bg: var(--colors-secondary);
  --button-text: var(--colors-secondary-foreground);
}
```

### 3. 动画效果
```css
.card {
  --transform-amount: 0deg;
  transform: rotate(var(--transform-amount));
  transition: --transform-amount 0.3s;
}

.card:hover {
  --transform-amount: 5deg;
}
```

## 最佳实践

### 1. 变量命名
- 使用语义化的名称
- 采用一致的命名模式
- 避免过深的嵌套

### 2. 变量组织
- 按功能分组
- 保持层级结构清晰
- 合理使用默认值

### 3. 性能优化
- 避免过多的变量嵌套引用
- 合理使用计算值
- 注意浏览器兼容性

## 常见问题与解决方案

### 1. 变量值未生效
可能的原因：
- 变量名称拼写错误
- 变量作用域不正确
- CSS选择器优先级问题

解决方案：
1. 仔细检查变量名称
2. 确认变量定义的位置
3. 检查CSS选择器优先级

### 2. 主题切换不生效
可能的原因：
- 主题类名应用位置错误
- 变量覆盖顺序问题
- JavaScript切换逻辑问题

解决方案：
1. 确保主题类名应用在正确的元素上
2. 检查CSS文件加载顺序
3. 验证主题切换的JavaScript代码
