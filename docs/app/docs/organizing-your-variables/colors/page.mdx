# Colors
颜色类变量的组织建议。

如果你对颜色类变量的组织和命名没有头绪，[shadcn/ui 的命名约定](https://ui.shadcn.com/docs/theming#convention)是一个很好的开始：
一般情况下，`background` 用于背景颜色，`foreground` 用于文本颜色，当 `background` 用于组件的背景颜色时，如 `colors/card/DEFAULT`[^1]，则省略 `background` 后缀。

{/* - `--background` - 背景色
- `--foreground` - 前景色/文本色
- `--muted` - 柔和的背景色
- `--muted-foreground` - 柔和的前景色
- `--popover` - 弹出层背景色
- `--popover-foreground` - 弹出层前景色
- `--card` - 卡片背景色
- `--card-foreground` - 卡片前景色
- `--border` - 边框色
- `--input` - 输入框边框色
- `--primary` - 主色调
- `--primary-foreground` - 主色调上的前景色
- `--secondary` - 次色调
- `--secondary-foreground` - 次色调上的前景色
- `--accent` - 强调色
- `--accent-foreground` - 强调色上的前景色
- `--destructive` - 危险/警告色
- `--destructive-foreground` - 危险/警告色上的前景色
- `--ring` - 聚焦环的颜色

<Callout type="warning" emoji="⚠️">
  以上是 CSS Variables，请不要在 Figma 中使用 `--` 作为任何变量的前缀。
</Callout> */}

{/* 对应到 Figma 中，可以使用以下命名约定： */}
- `colors/background/DEFAULT` - 背景色  
- `colors/foreground/DEFAULT` - 前景色
- `colors/muted/DEFAULT` - 柔和的背景色
- `colors/muted/foreground` - 柔和的前景色
- `colors/popover/DEFAULT` - 弹出层背景色
- `colors/popover/foreground` - 弹出层前景色
- `colors/card/DEFAULT` - 卡片背景色
- `colors/card/foreground` - 卡片前景色
- `colors/border/DEFAULT` - 边框色
- `colors/input/DEFAULT` - 输入框边框色
- `colors/primary/DEFAULT` - 主色调
- `colors/primary/foreground` - 主色调上的前景色
- `colors/secondary/DEFAULT` - 次色调
- `colors/secondary/foreground` - 次色调上的前景色
- `colors/accent/DEFAULT` - 强调色
- `colors/accent/foreground` - 强调色上的前景色
- `colors/destructive/DEFAULT` - 危险/警告色
- `colors/destructive/foreground` - 危险/警告色上的前景色
- `colors/ring/DEFAULT` - 聚焦环的颜色

如果你在 Figma 中按照上述命名约定组织变量，Variables Xporter 将导出以下结果：

## 对于Tailwind CSS 项目
如果你的项目使用 Tailwind CSS，在 Figma 中你的变量必须保证以 `colors` 作为顶层目录，因为在 Tailwind CSS 的配置文件中，只有 `colors` 配置键对于颜色类样式是有效的。
