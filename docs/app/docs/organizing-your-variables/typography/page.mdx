# 排版系统
让我们来探索如何组织字体和排版相关的变量，包括 `font-family`（字体系列）、`font-size`（字体大小）、`font-weight`（字重）和 `line-height`（行高）等。

## 顶层目录
为了保持变量结构的清晰和直观，且符合 Tailwind CSS 的配置结构，我们建议采用以下两种方式来组织字体相关变量：
### 使用样式名称作为顶层目录
- `font-family/{variable}`
- `font-size/{variable}` 
- `font-weight/{variable}` 
- `line-height/{variable}` 
- `letter-spacing/{variable}` 
- `line-clamp/{variable}`

例如：
- `font-weight/normal`
- `font-weight/medium`
- `font-weight/bold`

### 使用 `font` 作为顶层目录
为了让 Design Tokens 的变量结构更加清晰，可以将所有排版和字体相关的变量都放在 `font` 目录下，例如：
- `font/size/sm`
- `font/size/base`
- `font/size/lg`
- `font/weight/normal`
- `font/weight/medium`
- `font/weight/bold`
- `font/line-height/none`
- `font/line-height/tight`
- `font/line-height/relaxed`

<Callout type="error" emoji="🚨">
在 Tailwind CSS 项目中，遵循顶层目录的要求是必须的。
</Callout>

## 合并处理 [#merge-property]
你还可以选择将相关的属性以 `font/{variable}/{property}` 的形式组合在一起。这种方式特别适合管理具有固定样式组合的文本相关变量。

例如，Tailwind CSS 默认的 [Font Size 工具](https://tailwindcss.com/docs/font-size)由 `sm`、`base`、`lg` 等不同的级别组成，而每一个级别都包含默认的 `font-size`、`line-height` 两个属性，例如：

#### `sm`
- `font/size/sm`
- `font/line-height/sm`

#### `base`
- `font/size/base`
- `font/line-height/base`

#### `lg`
- `font/size/lg`
- `font/line-height/lg`

这是创建  `font-size` 的原子化变量的绝佳方式。

---

除了以上的原子化变量，还有一个典型的示例，就是将这些字体属性进一步抽象成 `display`、`heading`、`body` 等变量：

#### `display`
- `font/display/size`
- `font/display/weight`
- `font/display/line-height`
- `font/display/letter-spacing`

#### `heading`
- `font/heading/size`
- `font/heading/weight`
- `font/heading/line-height`
- `font/heading/letter-spacing`

#### `body`
- `font/body/size`
- `font/body/weight`
- `font/body/line-height`
- `font/body/letter-spacing`

这种组织方式会 Variables Xporter 会自动生成以下 Tailwind CSS 配置和对应的 CSS 变量：

```js
/** @type {import('tailwindcss').Config} */
module.exports = {
  theme: {
    fontSize: {
      'display': ['var(--font-display-size)', {
        lineHeight: 'var(--font-display-line-height)',
        letterSpacing: 'var(--font-display-letter-spacing)',
        fontWeight: 'var(--font-display-weight)',
      }],
      'heading': ['var(--font-heading-size)', {
        lineHeight: 'var(--font-heading-line-height)',
        letterSpacing: 'var(--font-heading-letter-spacing)',
        fontWeight: 'var(--font-heading-weight)',
      }],
      'body': ['var(--font-body-size)', {
        lineHeight: 'var(--font-body-line-height)',
        letterSpacing: 'var(--font-body-letter-spacing)',
        fontWeight: 'var(--font-body-weight)',
      }],
    }
  }
}
```
```css
:root {
  --font-display-size: 1.5rem;
  --font-display-line-height: 2rem;
  --font-display-letter-spacing: -0.01em;
  --font-display-weight: 500;

  --font-heading-size: 1.25rem;
  --font-heading-line-height: 1.75rem;
  --font-heading-letter-spacing: -0.01em;
  --font-heading-weight: 500;

  --font-body-size: 1rem;
  --font-body-line-height: 1.5rem;
  --font-body-letter-spacing: -0.01em;
  --font-body-weight: 400;
}

```
<Callout type="warning" emoji="⚠️">
需要注意的是，合并处理功能仅支持四种样式属性：`font-size`、`font-weight`、`line-height` 和 `letter-spacing`。另外，只有在存在 `font/{variable}/size` 变量时，这个功能才会生效。这是为了与 Tailwind CSS 的 [`fontSize` 配置功能](https://tailwindcss.com/docs/font-size#customizing-your-theme)保持一致。
</Callout>
