# 排版系统
让我们来探索如何组织字体和排版相关的变量，包括 `font-family`（字体系列）、`font-size`（字体大小）、`font-weight`（字重）和 `line-height`（行高）等。

## 顶层目录
为了保持变量结构的清晰和直观，且符合 Tailwind CSS 的配置结构，请按照以下规则命名：
### Tailwind CSS V4 命名规则
- `font/{variable}` -> **font-family**
- `text/{variable}` -> **font-size**
- `font-weight/{variable}` -> **font-weight**
- `leading/{variable}` -> **line-height**
- `tracking/{variable}` -> **letter-spacing**

例如：
- `text/sm`
- `font/sans`
- `font-weight/bold`

### Tailwind CSS V3 命名空间规则

- `font-family/{variable}`
- `font-size/{variable}` 
- `font-weight/{variable}` 
- `line-height/{variable}` 
- `letter-spacing/{variable}` 

例如：
- `font-weight/normal`
- `font-weight/medium`
- `font-weight/bold`


<Callout emoji="💡">
你可以任意选择以上两种命名规则，Variables Xporter 都能兼容，并根据导出的格式自动调整为对应 Tailwind CSS 配置的命名规则，但在以下的文档中，我们默认使用 Tailwind CSS V4 的命名规则。
</Callout>

## 合并处理 [#merge-property]
你还可以选择将相关的属性以 `text/{variable}/{property}` 的格式组合在一起。这种方式特别适合管理具有固定样式组合的文本相关变量。

例如，Tailwind CSS 默认的 [Font Size 工具](https://tailwindcss.com/docs/font-size)由 `sm`、`base`、`lg` 等不同的级别组成，而每一个级别除了包含默认的 `font-size` 属性，还包含 `line-height` 属性，例如：

#### `sm`
- `text/sm/DEFAULT`
- `text/sm/line-height`

#### `base`
- `text/base/DEFAULT`
- `text/base/line-height`

#### `lg`
- `text/lg/DEFAULT`
- `text/lg/line-height`

这是创建  `font-size` 的原子化变量的绝佳方式。

---

除了以上的原子化变量，还有一个典型的示例，就是将这些字体属性进一步抽象成 `display`、`heading`、`body` 等变量：

#### `display`
- `text/display/DEFAULT`
- `text/display/font-weight`
- `text/display/line-height`
- `text/display/letter-spacing`

#### `heading`
- `text/heading/DEFAULT`
- `text/heading/font-weight`
- `text/heading/line-height`
- `text/heading/letter-spacing`

#### `body`
- `text/body/DEFAULT`
- `text/body/font-weight`
- `text/body/line-height`
- `text/body/letter-spacing`

这种组织方式 Variables Xporter 会自动生成以下 Tailwind CSS 配置和对应的 CSS 变量：

<Callout type="warning" emoji="⚠️">
需要注意的是，合并处理功能仅支持四种样式属性 font-size、font-weight、line-height 和 letter-spacing。另外，只有在存在 font-size 属性时，这个功能才会生效。这是为了与 Tailwind CSS 的 [`fontSize` 配置功能](https://tailwindcss.com/docs/font-size#customizing-your-theme)保持一致。
</Callout>
