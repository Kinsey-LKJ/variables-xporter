# 组织你的设计变量 🎨

嗨！👋 在设计系统中，变量的组织方式至关重要。合理的组织不仅能提升设计和开发的效率，对于使用 Tailwind CSS 的项目来说，更是决定了设计令牌能否被正确导出和使用。让我们一起学习如何组织这些变量吧！

## 快速上手

<Steps>
### 创建变量集合
在 Figma 中创建新的变量集合（Collection），根据类型分类你的变量。

### 规划变量结构
按照类型（颜色、排版、间距等）组织变量，确保结构清晰。

### 遵循命名规范
使用一致的命名方式，推荐使用 kebab-case 或 camelCase。

### 设置变量类型
为每个变量选择正确的类型（颜色、数字等），这将影响导出结果。
</Steps>

## 为什么变量组织很重要

- **可维护性**：清晰的变量结构让团队成员能够轻松找到和更新所需的设计令牌
- **可扩展性**：良好的组织方式为未来的设计系统扩展提供了坚实的基础
- **一致性**：统一的命名和分类确保了设计语言在整个产品中的一致性
- **效率**：合理的变量组织能够加速设计和开发的工作流程
- **Tailwind 兼容性**：变量结构需要尽可能与 Tailwind CSS 的配置结构保持一致，确保正确的导出和映射

## 实用建议

<Callout type="info" emoji="💡">
  让设计系统更出色的秘诀就是良好的变量组织！以下建议将帮助你建立一个易于维护、随时可以扩展的变量系统。
</Callout>

### 命名和结构

<Steps>
### 选择命名规范
根据团队习惯选择 kebab-case 或 camelCase，保持一致性。

### 规划层级结构
```
color/
  ├─ primary/
  │   ├─ default
  │   ├─ hover
  │   └─ active
  └─ text/
      ├─ primary
      └─ secondary
```

### 设置变量类型
确保选择正确的变量类型，这会影响导出结果：
- 颜色类型 → Tailwind colors
- 数字类型 → spacing, fontSize 等
- 字符串类型 → fontFamily 等

### 添加变量描述
为重要变量添加清晰的描述，帮助团队理解其用途。
</Steps>

### 最佳实践

- 定期检查和优化变量组织
- 记录重要变量的使用场景
- 使用集合（Collection）归类变量
- 确保变量名称在导出时能被正确解析
- 避免过深的嵌套层级（建议不超过 3 层）
- 保持变量结构与 Tailwind 配置的一致性

<Callout type="warning">
  注意：变量名称中的特殊字符可能会影响导出结果。建议仅使用字母、数字、连字符和下划线。
</Callout>
