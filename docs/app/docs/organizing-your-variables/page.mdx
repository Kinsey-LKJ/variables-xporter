# 组织你的设计变量

在设计系统中，变量的组织方式至关重要。它不仅能提升设计和开发的效率，对于使用 Tailwind CSS 的项目来说，更是决定了设计令牌能否被正确导出和使用。合理的变量组织能让设计师更轻松地管理系统，也能让开发者更方便地应用这些设计决策。

# 为什么变量组织很重要

- **可维护性**：清晰的变量结构让团队成员能够轻松找到和更新所需的设计令牌
- **可扩展性**：良好的组织方式为未来的设计系统扩展提供了坚实的基础
- **一致性**：统一的命名和分类确保了设计语言在整个产品中的一致性
- **效率**：合理的变量组织能够加速设计和开发的工作流程
- **Tailwind 兼容性**：变量结构需要与 Tailwind CSS 的配置结构保持一致，确保正确的导出和映射

## 实用建议

以下是一些可以帮助你更好地组织变量的建议：

- 根据团队习惯选择合适的命名方式
- 按照项目需求组织变量结构
- 遵循 Tailwind CSS 的配置结构（这是必须的）
- 为变量添加适当的描述
- 定期检查和优化变量组织
- 记录重要变量的使用场景
- 使用集合（Collection）归类变量
- 确保变量名称在导出时能被正确解析
