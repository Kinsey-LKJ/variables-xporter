# Tailwind CSS V3

When choosing the Tailwind CSS export mode, Variables Xporter will generate two files:

- `tailwind.config.js` - Tailwind CSS configuration file
- `global.css` - Contains all CSS variable definitions

## Features and Optimizations

### 1. Color Value Optimization

Automatically converts color values to RGB channel format, supporting Tailwind's opacity modifiers:

```js
// tailwind.config.js
colors: {
  primary: 'rgb(var(--colors-primary-default))'
}

// global.css
:root {
  --colors-primary-default: 192 57 57;  /* RGB channel format */
}
```

Usage example:

```html
<div class="text-primary/80">
  <!-- 80% opacity -->
  Hello World
</div>
```

### 2. Typography Style Merging

Automatically merges related font variables, following Tailwind's conventions:

```js
fontSize: {
  'heading-1': ['2.5rem', { lineHeight: '1.2' }]
}
```

### 3. Multi-mode Support

Through **variable tracing** and the **Single Outlet Principle**, Variables Xporter can implement complex multi-mode variable exports.

```css
:root {
  --colors-primary: var(--colors-primary-light);
}

.dark {
  --colors-primary: var(--colors-primary-dark);
}
```

### 4. Powerful Compatibility
Variables Xporter supports both Tailwind CSS V3 and V4 export modes, and one set of Figma variables can seamlessly switch between the two export modes without any changes. Variables Xporter will automatically adjust the export based on different version naming conventions.
