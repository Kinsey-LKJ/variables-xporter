# Merge Typography Styles

Merge Typography Styles is a feature specifically designed for Tailwind CSS that intelligently combines font-related variables into configurations that follow Tailwind's conventions.

## How It Works

When you define font-related variables in Figma (such as font size, line height, etc.), Variables Xporter will:

1. Identify related variables
2. Combine variables according to naming rules
3. Generate Tailwind CSS configuration

### Example Conversion

Figma variables:
- `font-size/heading-1/DEFAULT`
- `font-size/heading-1/line-height`
- `font-size/heading-1/font-weight`

Converted to Tailwind CSS configuration or CSS variables:

## Naming Conventions

For Variables Xporter to correctly identify and merge variables, please follow these naming rules:

1. **Basic Naming Format**

`font-size/{variable}/{property}`


2. **Supported Properties**
   - `font-size`
   - `line-height`
   - `font-weight`
   - `letter-spacing`

3. **Example Names**

- `font-size/sm/DEFAULT`
- `font-size/sm/line-height`
- `font-size/sm/font-weight`
- `font-size/sm/letter-spacing`

<Callout type="warning" emoji="⚠️">
Note that the merge functionality only supports four style properties: font-size, font-weight, line-height, and letter-spacing. Additionally, this feature only works when a font-size property exists. This is to maintain consistency with Tailwind CSS's [`fontSize` configuration feature](https://tailwindcss.com/docs/font-size#customizing-your-theme).
