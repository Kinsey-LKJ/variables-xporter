# Export de Thème shadcn/ui

Variables Xporter s'intègre parfaitement avec shadcn/ui, exportant directement les Variables Figma en tant que configurations de thème shadcn/ui.

## Commencer

### 1. Vérifier la Convention de Nommage

Pour une intégration transparente, vos Variables Figma doivent suivre les conventions de nommage de shadcn/ui. Voici la correspondance complète entre les variables de thème shadcn/ui et les noms de Variables Figma que vous devriez créer :

| Thème shadcn/ui          | Nommage Variable Figma                |
| :------------------------ | :--------------------------------- |
| `--background`            | `colors/background/DEFAULT` [^1]   |
| `--foreground`            | `colors/foreground/DEFAULT`        |
| `--card`                  | `colors/card/DEFAULT`              |
| `--card-foreground`       | `colors/card/foreground`           |
| `--popover`               | `colors/popover/DEFAULT`           |
| `--popover-foreground`    | `colors/popover/foreground`        |
| `--primary`               | `colors/primary/DEFAULT`           |
| `--primary-foreground`    | `colors/primary/foreground`        |
| `--secondary`             | `colors/secondary/DEFAULT`         |
| `--secondary-foreground`  | `colors/secondary/foreground`      |
| `--muted`                 | `colors/muted/DEFAULT`             |
| `--muted-foreground`      | `colors/muted/foreground`          |
| `--accent`                | `colors/accent/DEFAULT`            |
| `--accent-foreground`     | `colors/accent/foreground`         |
| `--destructive`           | `colors/destructive/DEFAULT`       |
| `--border`                | `colors/border/DEFAULT`            |
| `--input`                 | `colors/input/DEFAULT`             |
| `--ring`                  | `colors/ring/DEFAULT`              |
| `--chart-1`               | `colors/chart-1/DEFAULT`           |
| `--chart-2`               | `colors/chart-2/DEFAULT`           |
| `--chart-3`               | `colors/chart-3/DEFAULT`           |
| `--chart-4`               | `colors/chart-4/DEFAULT`           |
| `--chart-5`               | `colors/chart-5/DEFAULT`           |
| `--sidebar`               | `colors/sidebar/DEFAULT`           |
| `--sidebar-foreground`    | `colors/sidebar/foreground`        |
| `--sidebar-primary`       | `colors/sidebar/primary`           |
| `--sidebar-primary-foreground` | `colors/sidebar/primary-foreground` |
| `--sidebar-accent`        | `colors/sidebar/accent`            |
| `--sidebar-accent-foreground` | `colors/sidebar/accent-foreground` |
| `--sidebar-border`        | `colors/sidebar/border`            |
| `--sidebar-ring`          | `colors/sidebar/ring`              |
| `--radius`                | `radius/radius`                    |

<Callout type="info">
  Pour les variables non-shadcn/ui, Variables Xporter les exportera comme configuration Tailwind CSS.
</Callout>

### 2. Sélectionner le Format Correspondant

Ouvrez Variables Xporter et sélectionnez shadcn/ui et votre version Tailwind CSS souhaitée dans le format d'exportation.
<figure className="mt-6 max-w-[400px]">
  <>![Sélectionner le format shadcn/ui](/shadcn-ui/choose-shadcn-ui-format.png)</>
  <figcaption>
    Sélectionner le format shadcn/ui
  </figcaption>
</figure>

### 3. Exporter

Sélectionnez les collections et groupes de variables que vous souhaitez exporter, cliquez sur le bouton d'exportation, et Variables Xporter exportera automatiquement vos Variables Figma en tant que configuration de thème shadcn/ui.
<figure className="mt-6 max-w-[400px]">
  <>![Résultat d'exportation](/shadcn-ui/export-successful.png)</>
  <figcaption>
    Résultat d'exportation
  </figcaption>
</figure>

### 4. Utiliser dans shadcn/ui

Copiez le fichier de configuration exporté dans le fichier de configuration Tailwind CSS de votre projet shadcn/ui et ajustez selon vos besoins.
