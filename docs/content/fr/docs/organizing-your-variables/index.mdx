# Organisez correctement vos Variables Figma

Des Variables Figma bien organisées sont essentielles pour une qualité d'exportation optimale et des systèmes de design maintenables. Nous fournissons des principes d'organisation, y compris le [Principe de Sortie Unique](/docs/organizing-your-variables/single-outlet-principle), pour gérer efficacement les variables multi-modes complexes.

## Points clés d'organisation

### Conception des collections

Organisez les structures de collection selon le [Principe de Sortie Unique](/docs/organizing-your-variables/single-outlet-principle) :

- **Collection principale** : Contient toutes les variables fournies en externe, servant de point de sortie unifié
- **Collections de modes** : Stockent uniquement les variables qui doivent changer selon les différents modes
- **Relations de référence** : Connectent les variables correspondantes entre la collection principale et les collections de modes via des références

### Conventions de nommage

- Suivez strictement la [structure de nommage](/docs/organizing-your-variables/principles/#naming-structure) de configuration de Tailwind CSS
- Utilisez le slash `/` pour le regroupement des variables
- Utilisez correctement le mot-clé `DEFAULT` pour les valeurs de base, facilitant une hiérarchie claire des variables

### Conseils pratiques

- Vérifiez régulièrement la structure des variables pour garantir des relations de référence claires
- Ajoutez des descriptions aux variables importantes pour la compréhension de l'équipe
- Évitez les structures imbriquées trop complexes
- Maintenez la cohérence et la prévisibilité dans la nomenclature

En adoptant ces stratégies d'organisation, vous pouvez gérer efficacement des Design Tokens complexes et construire un système de design à la fois puissant et maintenable.
