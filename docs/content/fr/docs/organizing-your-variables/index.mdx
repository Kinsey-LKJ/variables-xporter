# Organiser vos variables de design

Face aux besoins croissants de gestion des Design Tokens, Variables Xporter recommande d'utiliser le [principe de point de sortie unique](/docs/single-outlet-principle) pour organiser votre système de variables. Une organisation judicieuse des variables améliore non seulement la qualité du code exporté, mais rend également les systèmes de design multi-modes complexes plus clairs et faciles à gérer.

## Points clés d'organisation

### Conception des collections

Organisez la structure des collections selon le [principe de point de sortie unique](/docs/single-outlet-principle) :

- **Collection principale** : contient toutes les variables exposées, servant de point de sortie unifié
- **Collections de modes** : stockent uniquement les variables qui doivent changer selon les différents modes
- **Relations de référence** : connectent les variables correspondantes entre la collection principale et les collections de modes par référencement

### Conventions de nommage

- Suivre strictement la structure des clés de configuration de Tailwind CSS
- Utiliser la barre oblique `/` pour le regroupement des variables
- Utiliser correctement le mot-clé `DEFAULT` pour les valeurs de base, facilitant l'établissement d'une hiérarchie claire des variables

### Conseils pratiques

- Vérifier régulièrement la structure des variables pour assurer des relations de référence claires
- Ajouter des descriptions aux variables importantes pour la compréhension de l'équipe
- Éviter les structures imbriquées trop complexes
- Maintenir la cohérence et la prévisibilité dans le nommage

En adoptant ces stratégies d'organisation, vous pouvez gérer efficacement les Design Tokens complexes et construire un système de design à la fois puissant et maintenable.
