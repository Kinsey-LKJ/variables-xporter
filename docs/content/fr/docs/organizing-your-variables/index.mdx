# Organiser vos variables de design

Face aux besoins de plus en plus complexes de gestion des Design Tokens, Variables Xporter recommande d'utiliser le [principe de sortie unique](/docs/single-outlet-principle) pour organiser votre système de variables. Une organisation appropriée des variables améliore non seulement la qualité du code exporté, mais rend également les systèmes de design multi-modes complexes clairs et gérables.

## Éléments essentiels d'organisation

### Conception des collections

Organiser la structure des collections selon le [principe de sortie unique](/docs/single-outlet-principle) :

- **Collection principale** : Contient toutes les variables fournies externellement, servant de sortie unifiée
- **Collections de mode** : Stockent seulement les variables qui doivent changer dans différents modes
- **Relations de référence** : Connectent les variables de la collection principale et des collections de mode par des références

### Conventions de nommage

- Suivre strictement la structure des clés de configuration de Tailwind CSS
- Utiliser la barre oblique `/` pour le regroupement des variables
- Utiliser correctement le mot-clé `DEFAULT` pour les valeurs de base, facilitant l'établissement d'une hiérarchie claire des variables

### Conseils pratiques

- Vérifier régulièrement la structure des variables pour assurer des relations de référence claires
- Ajouter des descriptions aux variables importantes pour la compréhension de l'équipe
- Éviter les structures imbriquées trop complexes
- Maintenir la cohérence et la prévisibilité dans le nommage

En adoptant ces stratégies d'organisation, vous pouvez gérer efficacement les Design Tokens complexes et construire un système de design à la fois puissant et maintenable.
