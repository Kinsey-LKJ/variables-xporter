# Autres variables de design

import { Callout } from 'nextra/components'

Dans un système de design, outre les couleurs, la typographie et l'espacement, il existe de nombreuses autres variables de design importantes. Examinons comment organiser ces variables pour maintenir la cohérence et la maintenabilité du système de design.

## Autres styles

### Points de rupture responsifs

Utilisez `breakpoint` ou `screens` comme répertoire de premier niveau :

- `breakpoint/sm`
- `breakpoint/md`
- `breakpoint/lg`

- `screens/sm`
- `screens/md`
- `screens/lg`

### `border-radius`

Utilisez `border-radius` ou `radius` comme répertoire de premier niveau :
- `border-radius/sm`
- `border-radius/md`
- `border-radius/lg`

- `radius/sm`
- `radius/md`
- `radius/lg`

### `filter:blur()`

Utilisez `blur` comme répertoire de premier niveau :
- `blur/sm`
- `blur/md`
- `blur/lg`

Vous pouvez consulter la liste complète des clés de configuration dans la [référence de configuration du thème Tailwind CSS](https://tailwindcss.com/docs/theme#theme-variable-namespaces).

## Variables au niveau des composants

Pour les variables au niveau des composants, le répertoire de premier niveau doit toujours suivre les principes de ce document, mais le nom du composant doit être utilisé comme deuxième couche, par exemple :
- `colors/button/DEFAULT`
- `colors/button/foreground`
- `colors/button/border`

- `colors/input/DEFAULT`

- `colors/card/DEFAULT`
- `colors/card/foreground`
- `colors/card/border`
