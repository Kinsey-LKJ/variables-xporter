# 其他设计变量

import { Callout } from 'nextra/components'

在设计系统中，除了颜色、排版和间距之外，还有许多其他重要的设计变量。让我们来看看如何组织这些变量，以保持设计系统的一致性和可维护性。

## 其他样式

### 响应式断点

使用 `breakpoint` 或 `screens` 作为顶级目录：

- `breakpoint/sm`
- `breakpoint/md`
- `breakpoint/lg`

- `screens/sm`
- `screens/md`
- `screens/lg`

### `border-radius`

使用 `border-radius` 或 `radius` 作为顶级目录：
- `border-radius/sm`
- `border-radius/md`
- `border-radius/lg`

- `radius/sm`
- `radius/md`
- `radius/lg`

### `filter:blur()`

使用 `blur` 作为顶级目录：
- `blur/sm`
- `blur/md`
- `blur/lg`

您可以在 [Tailwind CSS 主题配置参考](https://tailwindcss.com/docs/theme#theme-variable-namespaces) 中查看完整的配置键列表。

## 组件级变量

对于组件级变量，顶级目录仍应遵循本文档中的原则，但组件名称应作为第二层，例如：
- `colors/button/DEFAULT`
- `colors/button/foreground`
- `colors/button/border`

- `colors/input/DEFAULT`

- `colors/card/DEFAULT`
- `colors/card/foreground`
- `colors/card/border`
