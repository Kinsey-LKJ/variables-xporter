# 组织你的设计变量

面对日益复杂的 Design Tokens 管理需求，Variables Xporter 推荐使用[单一出口原则](/docs/single-outlet-principle)来组织你的变量系统。合理的变量组织不仅能提升导出代码的质量，更能让复杂的多模式设计系统变得清晰易管理。

## 组织要点

### 集合设计

按照[单一出口原则](/docs/single-outlet-principle)组织集合结构：

- **主集合**: 包含所有对外提供的变量，作为统一出口
- **模式集合**: 只存储在不同模式下需要变化的变量
- **引用关系**: 通过引用连接主集合和模式集合的对应变量

### 命名规范

- 严格遵循 Tailwind CSS 的配置键结构
- 使用斜杠 `/` 进行变量分组
- 合理使用 `DEFAULT` 关键字表示基础值，便于建立清晰的变量层级

### 实用建议

- 定期检查变量结构，确保引用关系清晰
- 为重要变量添加描述，便于团队理解
- 避免过度复杂的嵌套结构
- 保持命名的一致性和可预测性

采用这些组织策略，您可以有效管理复杂的 Design Tokens，构建既强大又易于维护的设计系统。
