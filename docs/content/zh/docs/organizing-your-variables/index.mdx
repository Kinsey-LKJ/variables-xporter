## 关键组织要点

### 集合设计

根据 [单一出口原则](/docs/organizing-your-variables/single-outlet-principle) 组织集合结构：

- **主集合**：包含所有外部提供的变量，作为统一出口
- **模式集合**：仅存储需要在不同模式下变化的变量
- **引用关系**：通过引用连接主集合和模式集合中的对应变量

### 命名规范

- 严格遵循 Tailwind CSS 配置键结构
- 使用斜杠 `/` 进行变量分组
- 正确使用 `DEFAULT` 关键字表示基础值，便于清晰的变量层级

### 实用技巧

- 定期检查变量结构，确保引用关系清晰
- 为重要变量添加描述，便于团队理解
- 避免过于复杂的嵌套结构
- 在命名中保持一致性和可预测性

遵循这些组织策略，确保您的 Design Tokens 可维护、可扩展，并且每次都能完美导出。
