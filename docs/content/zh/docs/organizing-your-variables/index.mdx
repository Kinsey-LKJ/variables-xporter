# 合理的组织您的 Figma Variables

面对日益复杂的 Design Tokens 管理需求，Variables Xporter 推荐使用 [单一出口原则](/docs/single-outlet-principle) 来组织您的变量系统。一个结构良好的变量组织不仅能提升导出代码的质量，还能使复杂的多模式设计系统清晰易于管理。

## 关键组织要点

### 集合设计

根据 [单一出口原则](/docs/single-outlet-principle) 组织集合结构：

- **主集合**：包含所有外部提供的变量，作为统一出口
- **模式集合**：仅存储需要在不同模式下变化的变量
- **引用关系**：通过引用连接主集合和模式集合中的对应变量

### 命名规范

- 严格遵循 Tailwind CSS 配置键结构
- 使用斜杠 `/` 进行变量分组
- 正确使用 `DEFAULT` 关键字表示基础值，便于清晰的变量层级

### 实用技巧

- 定期检查变量结构，确保引用关系清晰
- 为重要变量添加描述，便于团队理解
- 避免过于复杂的嵌套结构
- 在命名中保持一致性和可预测性

通过采用这些组织策略，您可以有效管理复杂的 Design Tokens，并构建一个既强大又易于维护的设计系统。
