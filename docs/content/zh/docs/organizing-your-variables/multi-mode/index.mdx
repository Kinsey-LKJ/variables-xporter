# 多模式变量组织

多模式是 Figma Variables 系统的强大功能，它允许同一个变量在不同的上下文中具有不同的值。合理地组织多模式变量是构建可扩展设计系统的关键。我们推荐使用[单一出口原则](/docs/single-outlet-principle)来管理复杂的多模式变量系统。

## 多模式应用场景

多模式变量系统适用于以下场景：
- **主题模式**: 明暗主题切换
- **品牌模式**: 多品牌产品支持  
- **设备模式**: 响应式设计适配
- **密度模式**: 界面紧凑度调整

## 组织策略

### 按维度分离集合

每个模式维度使用独立集合管理，配合[单一出口原则](/docs/single-outlet-principle)建立清晰的引用层次。

### 精简变量

模式集合只包含在该维度下确实需要变化的变量。

### 语义化命名

使用描述性的变量名和模式名，如 `colors/primary` 和 `light/dark`。

## 创建复杂的多模式变量系统

使用单一出口原则能够创建复杂多维度的变量系统，例如包含明暗主题、品牌主题、响应式设计以及紧凑性模式等复合模式的变量系统，让我们一步步创建，并体会单一出口原则带来的好处。

### 创建 Design Tokens 集合（主集合）

首先，我们创建一个主集合来存储所有设计变量：

```
┌────────────────────────┬────────────────────────┐
│                        │                        │
│ Design Tokens          │         default        │
│                        │                        │
├────────────────────────┼────────────────────────┤
│                        │                        │
│ colors/primary/DEFAULT │ #EE6D21                │
│ colors/zhiyun/600      │ #EE6D21                │
│ colors/zhiyun/500      │ #E55517                │
│ colors/fiveray/600     │ #DA5959                │
│ colors/fiveray/500     │ #C03939                │
│ ...                    │ ...                    │
│                        │                        │
└────────────────────────┴────────────────────────┘
```

这个主集合仅包含一个默认模式，并作为所有变量的统一出口。

### 创建主题模式集合

接下来，我们创建一个专门用于管理主题模式的集合：

```
┌────────────────────────┬──────────────────────┬─────────────────────┐
│                        │                      │                     │
│ Theme Modes            │         light        │         dark        │
│                        │                      │                     │
├────────────────────────┼──────────────────────┼─────────────────────┤
│                        │                      │                     │
│ colors/primary/DEFAULT │ colors/primary/light │ colors/primary/dark │
│                        │                      │                     │
└────────────────────────┴──────────────────────┴─────────────────────┘
```

这个集合专门用于存储需要在明暗主题之间切换的变量。

### 创建多维度的变量系统

我们目前已经拥有了一个支持明暗主题的变量系统，现在我们希望在明暗主题的基础上更进一步，加入品牌主题的变量系统。
创建一个 Brands Modes 集合，用于管理品牌主题的变量，并创建两个不同的品牌模式（本例中是 `zhiyun` 和 `fiveray`），再创建两个变量 `colors/primary/light` 和 `colors/primary/dark`，并在两个模式下设置不同的值（本例中引用了主集合 Design Tokens 中的相关颜色）。
```
 ┌────────────────────────┬──────────────────────┬─────────────────────┐
 │                        │                      │                     │
 │ Brands Modes           │        zhiyun        │        fiveray      │
 │                        │                      │                     │
 ├────────────────────────┼──────────────────────┼─────────────────────┤
 │                        │                      │                     │
 │ colors/primary/light   │ colors/zhiyun/600    │ colors/fiveray/600  │
 │                        │                      │                     │
 │ colors/primary/dark    │ colors/zhiyun/600    │ colors/fiveray/500  │
 │                        │                      │                     │
 │ ...                    │ ...                  │ ...                 │
 │                        │                      │                     │
 └────────────────────────┴──────────────────────┴─────────────────────┘
```
接下来建立变量的引用关系：Theme Modes 集合中的变量根据当前主题引用 Brands Modes 集合中对应的变量，而 Brands Modes 集合中的变量则引用 Design Tokens 集合中的具体颜色值。这样，当切换品牌或主题时，整个系统的颜色都会相应更新。

### 创建响应式设计模式集合

通常，字体尺寸变量会根据不同的屏幕尺寸进行调整，以 `font/hero/size` 这个变量为例，首先在 Design Tokens 集合中创建一个 `font/hero/size` 变量，并创建 `font/size/2xl` 和 `font/size/3xl` 两个变量。

### 创建紧凑性模式集合

有的B端后台项目可能会有紧凑性模式，这种模式下，各种尺寸可能都会变小，比如字体、间距、容器宽度等。让我们进一步扩展我们的变量系统，加入紧凑性模式。
首先，在 Design Tokens 集合中创建一系列的 `spacing` 变量，用于管理间距、尺寸。为了方便演示，我们只创建了 `spacing/1` 这一个变量，在实际的项目中，您需要创建更多的变量，比如 `spacing/2`、`spacing/3` `spacing/4` 等等。查看[Spacing 变量的组织建议](/docs/organizing-your-variables/spacing)。
