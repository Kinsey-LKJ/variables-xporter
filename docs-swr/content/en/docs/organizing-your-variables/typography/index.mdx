# Typography System

Let's explore how to organize font and typography-related variables, including `font-family`, `font-size`, `font-weight`, and `line-height`.

## Top-level Directory
To maintain a clear and intuitive variable structure that aligns with Tailwind CSS's configuration structure, please follow these naming rules:

### Tailwind CSS V4 Naming Rules
- `font/{variable}` -> **font-family**
- `text/{variable}` -> **font-size**
- `font-weight/{variable}` -> **font-weight**
- `leading/{variable}` -> **line-height**
- `tracking/{variable}` -> **letter-spacing**

For example:
- `text/sm`
- `font/sans`
- `font-weight/bold`

### Tailwind CSS V3 Naming Rules

- `font-family/{variable}`
- `font-size/{variable}` 
- `font-weight/{variable}` 
- `line-height/{variable}` 
- `letter-spacing/{variable}` 

For example:
- `font-weight/normal`
- `font-weight/medium`
- `font-weight/bold`

<Callout emoji="ðŸ’¡">
You can choose either naming convention above, Variables Xporter is compatible with both and will automatically adjust to the corresponding Tailwind CSS configuration naming rules based on the export format. However, in the following documentation, we'll use Tailwind CSS V4 naming rules by default.
</Callout>

## Merge Processing [#merge-property]
You can also choose to combine related properties in a `text/{variable}/{property}` format. This approach is particularly suitable for managing text-related variables with fixed style combinations.

For example, Tailwind CSS's default [Font Size utilities](https://tailwindcss.com/docs/font-size) consist of different levels like `sm`, `base`, `lg`, etc., and each level includes not only the default `font-size` property but also the `line-height` property:

#### `sm`
- `text/sm/DEFAULT`
- `text/sm/line-height`

#### `base`
- `text/base/DEFAULT`
- `text/base/line-height`

#### `lg`
- `text/lg/DEFAULT`
- `text/lg/line-height`

This is an excellent way to create atomic font-size variables.

---

Beyond these atomic variables, another typical example is further abstracting these font properties into variables like `display`, `heading`, `body`:

#### `display`
- `text/display/DEFAULT`
- `text/display/font-weight`
- `text/display/line-height`
- `text/display/letter-spacing`

#### `heading`
- `text/heading/DEFAULT`
- `text/heading/font-weight`
- `text/heading/line-height`
- `text/heading/letter-spacing`

#### `body`
- `text/body/DEFAULT`
- `text/body/font-weight`
- `text/body/line-height`
- `text/body/letter-spacing`

This organization method will automatically generate the following Tailwind CSS configuration and corresponding CSS variables in Variables Xporter:
