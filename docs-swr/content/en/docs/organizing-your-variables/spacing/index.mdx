# Spacing System

Spacing and sizing are key elements in a design system, helping us create consistent visual hierarchy and layout rhythm. Let's explore how to organize these important design variables.

## Top-level Directory
Please use `spacing` as the top-level directory for all spacing and sizing variables. Here are some common naming examples:

Semantic naming:
- `spacing/sm` - Small spacing, suitable for compact element layouts
- `spacing/md` - Medium spacing, suitable for regular element spacing
- `spacing/lg` - Large spacing, suitable for section separation
- `spacing/xl` - Extra large spacing, suitable for major area separation

Numeric naming (atomic variables):
- `spacing/0` - No spacing (0px)
- `spacing/1` - Minimum spacing (typically 4px)
- `spacing/2` - Double minimum spacing (typically 8px)
- `spacing/4` - Quadruple minimum spacing (typically 16px)

## Unit Conversion

Variables Xporter provides intelligent unit conversion functionality. You can use the more intuitive px units in Figma for design, then choose the "Use Rem Units" option during export, and the tool will automatically convert them to accessibility-friendly rem units.
